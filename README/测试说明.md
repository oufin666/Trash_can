## 四、PID相关宏定义说明
oyq说：这一步是调参前的准备工作，需要先把宏定义设置好才可以去调速度pid；还有就是你得把四个电机接对，转向要确定好啊，具体看引脚分配去接，如果接上发现转向不对就去motor.c里面改一下方向就好

### 1. 宏定义位置
所有与PID闭环控制相关的宏定义位于 `Drivers/BSP/Inc/motor.h` 文件中。

### 2. 宏定义详解

#### 2.1 电机基础参数（**需根据实际硬件修改**）
oyq说：你主要改的就是这里，其他基本都不用改

**重要**：在开始PID调参前，必须先确认以下硬件参数并修改对应的宏定义，否则速度计算会不准确！
| 参数 | 宏定义 | 获取方式 | 检查方法 |
|------|--------|----------|----------|
| 电机最大转速 | `MOTOR_MAX_SPEED_PPS` | 电机说明书 | 运行电机至最高速，观察串口输出 |
oyq:把车放地上，车整体前进，四个电机都最大速度前进看一下就知道了，注意这个要带载去测试，不可以空转！
| 电机减速比 | `MOTOR_GEAR_RATIO` | 电机说明书 | 手动转动电机轴，数输出轴转数 |
| 编码器线数 | `ENCODER_RESOLUTION` | 编码器说明书 | 见下文检查方法 |
| 车轮直径 | `WHEEL_DIAMETER` | 实际测量 | 使用尺子测量车轮外径 |

车轮转一圈的累计脉冲数 = 减速比 * 编码器线数 * 倍频
如果这一点不符合，就应该去改倍频

**`MOTOR_MAX_SPEED_PPS` 计算公式**：
MAX_SPEED_PPS = (电机额定RPM / 60) × ENCODER_PPR × MOTOR_GEAR_RATIO
              = (300 / 60) × 52 × 30
              = 7800 pps

调试模式切换（给你看累计脉冲的，意思就是你切到那个看累计脉冲数的模式去看一下倍频是否正确，再切回正常的速度pid模式去调参）：
**功能说明**：通过单个宏定义 `DEBUG_MODE` 控制串口输出内容，定义在 `Drivers/BSP/Inc/motor.h` 中：

```c
/* 调试输出模式选择 */
/* DEBUG_MODE = 0: 输出编码器累计脉冲数 */
/* DEBUG_MODE = 1: 输出速度PID信息（目标速度、实际速度）*/
#define DEBUG_MODE           1       /*!< 当前调试模式，默认1（速度PID模式）*/
```

**模式说明**：

| `DEBUG_MODE` 值 | 输出内容示例 | 用途 |
|-----------------|-------------|------|
| 0 | `ENC: 1:  123456 2:  123458 3:  123455 4:  123457` | 测试编码器接线和计数是否正确 |
| 1 | `T:500 \| 1:  495 2:  498 3:  493 4:  496 pps` | PID调参，观察速度跟随情况 |

**使用方法**：
1. 修改 `motor.h` 中 `DEBUG_MODE` 的值（0或1）
2. 重新编译下载程序
3. 串口3将输出对应模式的信息

## 五、速度PID调参说明

### 1. PID参数位置
PID参数定义在 `Drivers/BSP/Src/pid.c` 文件中，每个电机独立配置：

```c
// 电机1 PID参数
PID_Position_Init(&g_PidMotor1, 5.0f, 0.5f, 1.0f);   // KP=5.0, KI=0.5, KD=1.0
PID_Position_SetLimits(&g_PidMotor1, 1000.0f, -1000.0f);//PID限幅
PID_Position_SetIntegralLimits(&g_PidMotor1, 300.0f, -300.0f);//积分限幅
```

### 2. 函数参数详解

#### 2.1 `PID_Position_Init(pid, KP, KI, KD)`

| 参数 | 类型 | 含义 | 当前值 | 调整建议 |
|------|------|------|--------|----------|
| **KP** | float | 比例系数 | 5.0 | 越大响应越快，过大会振荡 |
| **KI** | float | 积分系数 | 0.5 | 消除稳态误差，过大易超调 |
| **KD** | float | 微分系数 | 1.0 | 抑制振荡，过大易受噪声影响 |

**调参技巧**：
- 先调KP：从0开始逐步增大，直到电机出现轻微振荡后回调20%
- 再加KI：从0开始逐步增大，消除静态误差
- 最后加KD：从小值开始，通常为KP的1/5~1/2

#### 2.2 `PID_Position_SetLimits(pid, max_out, min_out)`

| 参数 | 类型 | 含义 | 当前值 | 调整建议 |
|------|------|------|--------|----------|
| **max_out** | float | PID输出上限 | 1000.0 | 对应电机最大PWM值 |
| **min_out** | float | PID输出下限 | -1000.0 | 通常设为对称负值 |

**说明**：限制发送给电机的PWM值范围，一般不需要修改。

#### 2.3 `PID_Position_SetIntegralLimits(pid, max_i, min_i)`

| 参数 | 类型 | 含义 | 当前值 | 调整建议 |
|------|------|------|--------|----------|
| **max_i** | float | 积分项上限 | 300.0 | 防止积分饱和 |
| **min_i** | float | 积分项下限 | -300.0 | 通常设为对称负值 |

**说明**：限制积分累积值，防止积分饱和导致超调，建议设为输出限幅的20%~40%。

### 3. 调参步骤
1. **初始化参数**：KP=0, KI=0, KD=0
2. **调比例**：逐步增大KP，直到电机出现轻微振荡，然后回调20%
3. **调积分**：逐步增大KI，消除稳态误差
4. **调微分**：适当加入KD，抑制振荡（通常为KP的1/5~1/2）
5. **微调优化**：反复调整三个参数达到最佳跟随效果

### 4. 速度单位说明
- **内部计算单位**：脉冲/秒（pps）
- **显示单位**：脉冲/秒（pps），串口输出格式：`T:500 | 1:  495 2:  498 3:  493 4:  496 pps`
- **单位转换**：脉冲/秒 × SPEED_CONV_FACTOR = 米/秒

### 5. 速度限幅
```c
#define MAX_SPEED_PPS       30000   // 最大速度限幅（防止int16_t溢出）
#define MIN_SPEED_PPS       (-MAX_SPEED_PPS)
```

### 6. 目标速度设置方法

**方法一：通过串口指令控制（推荐）**

使用串口3发送运动控制指令，自动设置对应方向的目标速度：

| 指令 | 指令码 | 目标速度设置 |
|------|--------|--------------|
| 前进 | `FF 01 FE` | 所有电机设置为 `DEFAULT_SPEED` |
| 后退 | `FF 02 FE` | 所有电机设置为 `-DEFAULT_SPEED` |
| 向左 | `FF 03 FE` | 按全向轮逻辑设置各电机速度 |
| 向右 | `FF 04 FE` | 按全向轮逻辑设置各电机速度 |
| 停止 | `FF 05 FE` | 所有电机速度设为 0 |

**方法二：修改默认速度宏定义**

在 `Drivers/BSP/Inc/motor.h` 中修改：

```c
#define DEFAULT_SPEED        500     /* 默认移动速度（脉冲/秒）*/
```

**速度范围说明**：
- 最大速度：`MOTOR_MAX_SPEED_PPS = 待你测试`（脉冲/秒）
- 速度单位：脉冲/秒（pps）

**调试输出格式**（DEBUG_MODE=1）：

```
T:1: 500 2:-500 3:-500 4: 500 | A:1: 495 2:-498 3:-493 4: 496 pps
```

- `T:1: 500`：电机1目标速度500 pps
- `A:1: 495`：电机1实际速度495 pps

### 7. 调参注意事项
- 调参时建议降低目标速度（如设为300-500 pps），避免高速下参数不当导致失控
- 观察串口输出的实际速度跟随情况，理想状态下实际速度应快速跟踪目标速度
- 注意电机正反转时参数对称性，必要时分别调整正反转参数
- 如果电机抖动严重，可适当减小KP和KD值

## 六、串口指令集

### 1. 指令格式

上位机发送指令格式：`FF（包头）+ 有效指令（一字节）+ FE（包尾）`

**示例**：发送前进指令 `FF 01 FE`

### 2. 运动控制指令

| 指令 | 指令码 | 功能 |
|------|--------|------|
| `CMD_MOVE_FORWARD` | `0x01` | 前进 |
| `CMD_MOVE_BACKWARD` | `0x02` | 后退 |
| `CMD_MOVE_LEFT` | `0x03` | 向左移动 |
| `CMD_MOVE_RIGHT` | `0x04` | 向右移动 |
| `CMD_MOVE_STOP` | `0x05` | 停止 |

### 3. PID调参指令

| 指令 | 指令码 | 功能 |
|------|--------|------|
| `CMD_KP_INC` | `0x11` | KP +1.0 |
| `CMD_KP_DEC` | `0x12` | KP -1.0 |
| `CMD_KI_INC` | `0x13` | KI +0.1 |
| `CMD_KI_DEC` | `0x14` | KI -0.1 |
| `CMD_KD_INC` | `0x15` | KD +1.0 |
| `CMD_KD_DEC` | `0x16` | KD -1.0 |

### 4. 全向轮底盘运动逻辑

**电机布局**（俯视）：

```
        电机1        电机2
          ↑           ↑
          |           |
    ┌─────┴─────┬─────┴─────┐
    │                       │
    │       底盘中心        │
    │                       │
    └─────┬─────┴─────┬─────┘
          │           │
          ↓           ↓
        电机4        电机3
```

### 5. 使用示例

```c
// 上位机发送示例（十六进制）：

// 前进
FF 01 FE

// 停止
FF 05 FE

// KP增加1
FF 11 FE

// KI减少0.1
FF 14 FE
```

### 6. 注意事项

- 指令通过串口3（USART3）接收
- 默认移动速度：`DEFAULT_SPEED = 500`（脉冲/秒）
- PID参数调整会同时作用于四个电机
- 指令解析具有容错机制，格式错误会自动忽略
