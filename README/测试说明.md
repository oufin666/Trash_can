# 测试说明

## 一、测试环境搭建

### 1. 硬件准备
- STM32F407开发板
- TB6612FNG电机驱动模块 × 2
- 直流减速电机 × 4
- LED指示灯（PC13）
- 串口调试助手
- ST-Link调试器

### 2. 软件准备
- STM32CubeIDE
- 串口调试助手软件

## 二、测试步骤

### 1. 编译下载
1. 在STM32CubeIDE中打开项目
2. 编译项目：点击 **Build** 按钮
3. 连接ST-Link调试器
4. 下载代码：点击 **Run** 按钮

### 2. 功能测试

#### 2.1 LED闪烁测试
- **测试方法**：观察开发板上的LED指示灯
- **预期结果**：LED每1秒闪烁一次，指示任务调度正常
## 指示灯正常


#### 2.2 串口通信测试
- **测试方法**：
  1. 连接USART2（PD5/PD6）和USART3（PD8/PB11）到电脑
  2. 打开串口调试助手，设置波特率115200
  3. 发送任意数据到两个串口
- **预期结果**：
  - **串口2**：接收到测试信息："USART2 Test Message"，发送的任何数据都会被原样回传
  - **串口3**：接收到测试信息："USART3 Test Message"，发送的任何数据都会被原样回传
## 串口2,3上电初均发送一次测试信息USARTX Test Message，之后串口2死掉，串口三以1HZ频率发编码器和ADC


#### 2.3 电机控制测试

#### 2.4 编码器测试
- **测试方法**：
  1. 观察串口3（调试/蓝牙串口）的输出
  2. 手动转动电机轴，观察转速变化
- **预期结果**：
  - **串口3**：每1秒输出一次电机转速，格式为：`1:XX.X  2:XX.X  3:XX.X  4:XX.X`（其中XX.X是电机转速，单位为RPM）
  - 手动转动电机时，对应电机的转速值会发生变化
  - 电机正转时转速为正值，反转时为负值

#### 2.5 ADC测试
- **测试方法**：
  1. 观察串口3（调试/蓝牙串口）的输出
  2. 连接PC1引脚到不同的电压源（0-3.3V）
- **预期结果**：
  - **串口3**：每1秒输出一次ADC采样值，格式为：`ADC：XXX`（其中XXX是12位ADC采样值，范围0-4095）
  - 当改变PC1引脚的输入电压时，ADC采样值会相应变化
  - 输入电压为0V时，采样值接近0；输入电压为3.3V时，采样值接近4095（12位ADC）
## ADC有微小误差但无妨，正常
